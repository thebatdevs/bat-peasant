# Context info prompt

Before answering, audit your current available context.

Return:

1. What project/repository do you think we are working in?
2. What task are we solving?
3. What files, rules, or instructions are currently important?
4. What architecture pattern should you follow?
5. What information are you missing?
6. Are you relying on exact context, summarized context, or assumptions?
7. Should I continue this thread or start a new one?

Be honest. If you cannot see something, say "not available in current context".
Do not guess.
